01 — Architecture

The transport layer for autonomous agents.

ACP defines the envelope, not the pipe. It runs over HTTP, MCP, or WebSocket. It requires an auth field but doesn't mandate a specific auth method — agAuth is the recommended layer.

02 — The Envelope

One format. Every transport.

Any ACP message — regardless of transport — uses the same envelope. Routing, auth, threading, and payload type in a single structure.

{
  "spec":         "acp/0.1",
  "id":           "msg-uuid",
  "from":         "ag-4f8a2c1b-000001-9e3d",
  "to":           "ag-7b3c9d2e-000042-ff1a",
  "timestamp":    "2026-02-12T00:00:00Z",
  "payload_type": "acp:invoke",
  "payload":      { },
  "auth":         "agtoken_here"
}
03 — Payload Types

Base vocabulary. Infinitely extensible.

ACP defines a base set of payload types. Services and agents extend with namespaced types: {namespace}:{action}

acp:ping
Liveness check
acp:pong
Response to ping
acp:discover
Request capability manifest
acp:manifest
Return capability manifest
acp:invoke
Invoke a declared capability
acp:result
Response to invocation
